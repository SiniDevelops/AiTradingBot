  API ENDPOINTS - IMPLEMENTED & TESTED

BASE URL: http://127.0.0.1:8000

[HEALTH CHECK]
GET /health
  Response: {"status": "ok", "timestamp": "ISO_TIMESTAMP"}
  
[INGESTION]
POST /ingest_news
  Request Body:
    {
      "id": "news_001",
      "source": "Bloomberg",
      "published_at": "2024-02-13T09:30:00Z",
      "title": "...",
      "content": "..."
    }
  Response:
    {
      "news_id": "news_001",
      "tickers": ["AAPL"],
      "status": "ingested|duplicate"
    }
  Steps:
    1. Clean text
    2. Hash for deduplication
    3. Extract tickers via ticker_linker
    4. Store in raw + clean tables
    5. Add profile chunks to vector store

[ANALYSIS]
POST /analyze_news/{news_id}
  Response: List of AnalyzeResponse
    [{
      "news_id": "news_001",
      "ticker": "AAPL",
      "analysis": {
        "ticker": "AAPL",
        "event_type": "earnings|lawsuit|guidance|product_launch|regulatory|macro|other",
        "is_new_information": true/false,
        "impact_score": -1.0 to 1.0,
        "horizon": "intraday|swing|long",
        "severity": "low|med|high",
        "confidence": 0.0 to 1.0,
        "risk_flags": ["rumor", "low_quality_source", "ambiguous", "already_priced_in"],
        "contradiction_flags": ["conflicts_with_guidance", "conflicts_with_state", "none"],
        "summary": "1-2 sentence summary",
        "evidence": "1 short excerpt",
        "citations": [
          {"layer": "profile|state|event", "source_id": "...", "why": "..."}
        ]
      },
      "audit_id": 1
    }]
  Steps:
    1. Load news and tickers
    2. For each ticker:
       a. Retrieve context (profile, state, events) via FAISS
       b. Call LLM analyzer with article + context
       c. Process analysis via StateManager (Option B)
       d. Rebuild and commit state snapshot
       e. Add to vector store (state layer)
       f. Store audit record

[STATE QUERY]
GET /state/{ticker}
  Response:
    {
      "ticker": "AAPL",
      "state": {
        "ticker": "AAPL",
        "open_events": [...],
        "recent_catalysts": [...],
        "key_risks": [...],
        "last_updated": "ISO_TIMESTAMP"
      }
    }
  Returns: Current state snapshot for ticker
           Open events + recent catalysts + key risks

[EVENT HISTORY]
GET /events/{ticker}?status=open|closed
  Response:
    {
      "ticker": "AAPL",
      "events": [
        {
          "id": 1,
          "ticker": "AAPL",
          "event_type": "earnings",
          "status": "open",
          "severity": "med",
          "impact_score": 0.5,
          "horizon": "swing",
          "summary": "...",
          "source_id": "news_001",
          "start_ts": "ISO_TIMESTAMP",
          "end_ts": null,
          "confidence": 0.85,
          "evidence": "...",
          "created_at": "ISO_TIMESTAMP"
        }
      ],
      "count": 1
    }
  Query Params:
    status (optional): "open" or "closed" to filter

[AUDIT]
GET /audit/{audit_id}
  Response:
    {
      "id": 1,

        }
      ],
      "llm_output": {
        "ticker": "AAPL",
        "event_type": "earnings",
        ...
      },
      "created_at": "ISO_TIMESTAMP"
    }
  Returns: Full audit trail with inputs, retrieval, and outputs

[BATCH ANALYSIS]
POST /batch_analyze

    ]
  Response:
    {
      "results": [
        {
          "news_id": "news_001",

        },
        ...
      ]
    }
  Returns: Results for multiple news articles

================================================================================
  TROUBLESHOOTING GUIDE
================================================================================

Issue: "ModuleNotFoundError: No module named 'pydantic'"
Solution: Activate venv: .\venv\Scripts\activate
Then: pip install -r requirements.txt

Issue: "Port 8000 already in use"
Solution: Use different port: 
  uvicorn app.main:app --host 127.0.0.1 --port 8001

Issue: "Database locked"
Solution: Delete and reinitialize:
  rm data/trading_bot.db
  python -c "from app import db; db.init_db()"

Issue: "FAISS not found / falling back to DB"
Solution: Optional - system works without it. To enable:
  pip install faiss-cpu

Issue: Unicode encoding errors in terminal
Solution: Use Python-escaped strings or ASCII equivalents
  Already fixed in demo.py

Issue: Script execution disabled (PowerShell)
Solution: Use direct Python path:
  .\venv\Scripts\python.exe -m uvicorn app.main:app ...
